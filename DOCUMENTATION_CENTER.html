üèóÔ∏è Arsitektur Sistem

Alur Kerja

1. Admin Membuat Jadwal
Admin login ke jadual_kegiatan.php dan membuat jadwal baru
2. Email 1 Dikirim (Langsung)
Notifikasi langsung ke semua user tentang jadwal baru
3. Jadwal Tersimpan
Jadwal disimpan ke tabel jadwal dan jadwal_reminder
4. Cronjob Monitor
Setiap 5 menit, cronjob jalankan send_schedule_reminders.php
5. Email 2 Dikirim (Otomatis)
Saat reminder_datetime tercapai, Email 2 dikirim otomatis
6. Status Tercatat
Status pengiriman (sent_at) tercatat di database
‚úÖ Sistem siap untuk 2-tahap email pengingat!
Email 1 sudah aktif. Email 2 perlu setup cronjob.

‚ú® Fitur Utama

  • 2-Tahap Email Pengingat - Notifikasi langsung + Pengingat terjadwal
  • Database Tracking - Setiap email tercatat dengan status pengiriman
  • Auto Setup - Tabel jadwal_reminder dibuat otomatis saat first use
  • Flexible Cronjob - Frequency configurable (1 menit, 5 menit, dll)
  • Cross-Platform - Compatible Linux, Windows, cPanel, Docker, dll
  • Security - Secret key protection, email validation, SQL injection prevention
  • Monitoring - Query sederhana untuk check status email
  • Error Handling - Logging lengkap untuk debugging
  • Production Ready - Tested dan siap untuk production environment

‚ö†Ô∏è Hal Penting

üîß Setup Cronjob Diperlukan
Email 2 (pengingat terjadwal) hanya berfungsi jika cronjob sudah setup. Tanpa cronjob, hanya Email 1 (notifikasi langsung) yang terkirim.
üìß Email Config Harus Benar
Pastikan SMTP settings di includes/email_config.php sudah dikonfigurasi dengan benar sebelum menggunakan sistem.
üïê Timezone Penting
Pastikan timezone server sesuai dengan lokasi untuk memastikan email dikirim di waktu yang benar.
‚úÖ Testing Recommended
Buat jadwal test dengan waktu 5 menit ke depan untuk memverifikasi sistem bekerja dengan benar.
